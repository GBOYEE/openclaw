## Reporting a Vulnerability

We take security seriously. To report a vulnerability, please email **security@example.com** (TODO: replace with actual contact). Do not open a public GitHub issue.

We will respond within 48 hours and aim to resolve within 90 days. Keep details confidential during the disclosure window.

### What to include
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested mitigations

We may award recognition for responsible disclosure depending on severity.

## Security Updates

Patches are released on the main branch. Keep your deployment up to date.

## Best Practices

- Run OpenClaw with least privileges
- Regularly update dependencies
- Use strong passwords and API keys
- Do not commit secrets
- Audit gateway configurations
